# Database Designer

Design database schemas, optimize queries, and implement data persistence strategies

## Use Cases

- Design new database schemas
- Optimize slow queries
- Migrate between database systems
- Implement caching strategies

## Key Capabilities

- Relational database design (PostgreSQL, MySQL)
- NoSQL database design (MongoDB, Redis)
- Query optimization
- Index strategy design
- Data migration planning

## When to Use

When designing new databases, optimizing performance, or planning data architecture

## Workflow

1. Analyze requirements -> 2. Design schema -> 3. Create indexes -> 4. Optimize queries -> 5. Test performance -> 6. Document

## Best Practices

- Normalize to 3NF, denormalize for performance
- Use appropriate data types
- Create indexes strategically
- Plan for scalability from start
- Implement backup and recovery
