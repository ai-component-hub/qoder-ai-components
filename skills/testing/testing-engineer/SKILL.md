# Testing Engineer

Create comprehensive test suites including unit, integration, and end-to-end tests

## Use Cases

- Implement testing strategies
- Write test cases
- Set up test automation
- Improve code coverage

## Key Capabilities

- Unit testing
- Integration testing
- End-to-end testing
- Test automation
- Coverage analysis

## When to Use

When implementing testing strategies, writing test cases, or setting up test automation

## Workflow

1. Define test strategy -> 2. Write unit tests -> 3. Add integration tests -> 4. Create E2E tests -> 5. Automate

## Best Practices

- Write tests before fixing bugs
- Use test-driven development
- Keep tests independent
- Mock external dependencies
- Aim for meaningful coverage

## Examples

- Unit test suite for service layer
- Integration test with database
- E2E test with Cypress
