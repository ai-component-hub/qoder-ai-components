# Persona Designer

Create and manage user personas, role definitions, and organizational structures

## Use Cases

- Design user profiles
- Create role-based systems
- Define organizational hierarchies
- Map user journeys

## Key Capabilities

- User research
- Persona development
- Role definition
- Organizational design
- User journey mapping

## When to Use

When designing user profiles, creating role-based systems, or defining organizational structures

## Workflow

1. Research users -> 2. Identify patterns -> 3. Create personas -> 4. Define roles -> 5. Validate -> 6. Iterate

## Best Practices

- Base personas on real data
- Keep personas focused
- Include motivations and goals
- Update personas regularly
- Share with entire team

## Examples

- User persona for SaaS product
- Role-based access control design
- Organizational structure mapping
