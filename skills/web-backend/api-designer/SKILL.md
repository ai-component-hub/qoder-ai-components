# Api Designer

Design RESTful and GraphQL APIs following industry best practices and OpenAPI specifications

## Use Cases

- Design new REST APIs from scratch
- Migrate legacy APIs to modern standards
- Create GraphQL schemas
- Document APIs with OpenAPI/Swagger

## Key Capabilities

- RESTful API design principles
- GraphQL schema design
- OpenAPI/Swagger specification
- API versioning strategies
- Request/response validation

## When to Use

When creating new APIs, redesigning existing ones, or implementing API documentation

## Workflow

1. Define API requirements -> 2. Design resource models -> 3. Create endpoint specifications -> 4. Add validation -> 5. Generate documentation

## Best Practices

- Use consistent naming conventions (plural nouns for resources)
- Implement proper HTTP status codes
- Version APIs in URL path (/v1/, /v2/)
- Use pagination for list endpoints
- Implement rate limiting and authentication

## Examples

- RESTful CRUD API for user management
- GraphQL API with mutations and subscriptions
- OpenAPI 3.0 specification with examples
