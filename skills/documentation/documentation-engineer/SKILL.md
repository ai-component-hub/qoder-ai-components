# Documentation Engineer

Create comprehensive, maintainable documentation following best practices

## Use Cases

- Write technical documentation
- Create API documentation
- Build architecture docs
- Develop user guides

## Key Capabilities

- Technical writing
- API documentation (OpenAPI, JSDoc)
- Architecture diagram creation
- User guide development
- Documentation automation

## When to Use

When writing technical documentation, API docs, architecture docs, or user guides. Execute first, document after.

## Workflow

1. Identify audience -> 2. Gather information -> 3. Create outline -> 4. Write content -> 5. Review -> 6. Publish

## Best Practices

- Write for the reader, not the writer
- Use consistent formatting
- Include examples
- Keep docs versioned with code
- Automate doc generation when possible

## Examples

- API reference documentation
- Architecture decision records
- User onboarding guide
