# Deployment Engineer

Create deployment configurations, CI/CD pipelines, and infrastructure-as-code

## Use Cases

- Set up CI/CD pipelines
- Deploy applications to production
- Create infrastructure-as-code
- Configure monitoring and alerting

## Key Capabilities

- Docker containerization
- Kubernetes orchestration
- CI/CD pipeline design (GitHub Actions, GitLab CI)
- Infrastructure-as-code (Terraform, Pulumi)
- Cloud deployment (AWS, GCP, Azure)

## When to Use

When deploying applications, setting up automation, or managing infrastructure

## Workflow

1. Design architecture -> 2. Containerize app -> 3. Create CI/CD -> 4. Set up infrastructure -> 5. Deploy -> 6. Monitor

## Best Practices

- Use infrastructure-as-code for everything
- Implement blue-green or canary deployments
- Automate rollback procedures
- Monitor everything
- Document deployment process

## Examples

- GitHub Actions CI/CD pipeline
- Kubernetes deployment manifests
- Terraform infrastructure setup
