# Documentation Standards.Md

**Category:** Documentation  
**Description:** Enforce documentation standards while avoiding documentation addiction. Focus on essential, actionable documentation.

---

## 🎯 Purpose

Ensure consistent documentation-standards.md across all projects

---

## 📏 Rules

- Maintain ONE source of truth
- Document APIs with OpenAPI/Swagger
- Create README for all projects
- Write deployment guides
- Create quick reference cards
- Avoid recursive documentation
- Update docs when code changes
- Use markdown formatting consistently

## ✅ Validation

- Check documentation-standards.md compliance in code review
