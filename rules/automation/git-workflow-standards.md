**Category:** Version  
**Description:** Enforce Git workflow standards including branching strategy, commit messages, and repository organization.

---

## 🎯 Purpose

Ensure consistent git-workflow-standards.md across all projects

---

## 📏 Rules

- Use conventional commit messages (feat:, fix:, docs:, etc.)
- Create feature branches from main/develop
- Require pull requests for all merges
- Keep commits atomic and focused
- Write descriptive commit messages
- Use .gitignore for all projects
- Never commit secrets or .env files
- Tag releases with semantic versioning
